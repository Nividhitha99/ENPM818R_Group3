## Retrospective

###  What Went Well
- **Service Progress**: All core services (uploader, processor, analytics) showed tangible progress with working features.
- **Infrastructure Validation**: Successfully validated EKS cluster and node provisioning, confirming the architecture works.
- **Cost Awareness**: Early identification of high AWS costs (NAT Gateway, nodes) allowed the team to address them proactively.
- **Frontend Integration**: Basic API integration started, showing the system coming together.

###  What Could Be Improved
- **Cost Planning**: Should have estimated AWS costs before provisioning to avoid surprises. NAT Gateway costs ($32/month per AZ) and node costs could have been anticipated.
- **IAM Issues**: Permission problems suggest earlier setup of IRSA (IAM Roles for Service Accounts) would have saved time.
- **Documentation**: Could have documented the manual EKS setup process as it was being done to help with troubleshooting.

###  Lessons Learned
- **AWS Costs Add Up Quickly**: NAT Gateways and EKS nodes are expensive. Always monitor costs and scale down when not in use.
- **Early Infrastructure Setup**: Setting up IAM roles and permissions early prevents blocking issues later.
- **Incremental Progress**: Seeing each service work independently before integration builds confidence.

### Action Items for Next Meeting
- Complete IAM role configuration for all services.
- Implement cost-saving measures (scale down nodes when idle).
- Complete GitHub Actions CI/CD pipeline.
- Test end-to-end workflow integration.
